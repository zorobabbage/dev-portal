＃＃ 介绍

虽然可以使用 Zilliqa 提供的公共端点与区块链交互，但我们建议所有希望支持主网上交易的交易所设置种子节点。本节将引导你完成启动和运行种子节点所需的基本步骤。

## 模式

可以通过 3 种不同的模式运行种子节点。交易所可以使用适合自身的任何模式。
- IP白名单
- 密钥白名单（有或者没有开放的入站端口）

### IP 白名单模式

在 IP 白名单模式下，区块链数据每隔一段时间直接推送到交易所。由于使用此模式的种子节点直接接收数据并且不会从对等方拉取数据，因此 Zilliqa 必须将交易所列入白名单才能接收这些数据广播。这需要一个静态的公共 IP 地址，至少有两个开放端口（入站和出站）可以访问它。 [IP 白名单](exchange-ip-whitelisting) 部分包含有关在 IP 白名单模式下运行节点的信息。

### 密钥白名单模式

在密钥白名单模式下，区块链数据由来自 Zilliqa Research 托管的公共种子节点的种子定期提取。使用这种模式的交易所会生成一个公私钥对，并与 Zilliqa Research 共享他们的公钥以用于白名单。必须至少为出站流量打开一个端口。使用此模式的交易所还可以选择为入站流量配置第二个端口。 [密钥白名单](exchange-key-whitelisting-1) 部分包含有关在密钥白名单模式下运行节点的信息。

## 最低硬件要求
- x64 Linux 操作系统（例如 Ubuntu 18.04.5）
- 最新的双核处理器 @ 2.2 GHZ。如：Intel Xeon (Skylake)
- 8GB DRR3 RAM 或更高版本
- 公共静态 IP 地址
- 500GB 固态硬盘
- 100MB/s 上传下载带宽
