在发送交易之前，必须首先使用 __合法私钥__ 对其进行签名。 这可以通过 Zilliqa 团队和社区提供的众多 SDK 之一来完成。

签名是针对交易内容的 Protobuf 序列化版本完成的。 这就是所有 SDK 都依赖 Protobuf 来运行的原因。 作为开发人员，这一步对你来说是 __透明的__。


提供私钥的示例：

```js
const { Zilliqa } = require('@zilliqa-js/zilliqa');
const { getAddressFromPrivateKey } = require('@zilliqa-js/crypto');
const zilliqa = new Zilliqa("https://dev-api.zilliqa.com");

const PRIVATE_KEY = '9afc1a1dab96127e902daaaec1a56c30346f007523c787c3bb62371c0e5a1be7'

zilliqa.wallet.addByPrivateKey(PRIVATE_KEY);
```
