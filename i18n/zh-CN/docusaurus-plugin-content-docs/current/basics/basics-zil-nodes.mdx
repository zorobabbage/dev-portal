Zilliqa 主网由多种类型的节点组成，这些节点执行不同的功能或提供专门的服务。

## 分片节点

分片节点都是 Zilliqa 网络挖矿节点。它们被这样命名，是因为它们被分配给网络内的若干个簇（即，分片）中的一个。属于同一分片中的节点处理同一组分配给他们的分片交易。处理后，节点必须以产生分片的微块实现对交易集的共识。 对于他们的工作贡献，分片节点在每个 DS 时期结束时都会获得 $ZIL 奖励。

在分片节点成功提交工作量证明 (PoW) 问题的解决方案后，分片成员资格由 DS 委员会确定，该问题在每个 DS 时期发生一次。

## DS 节点

DS（目录服务）节点也是挖矿节点，具有以下额外职责：（1）聚合分片微块以形成交易块（2）在每个 DS 纪元开始时创建 DS 块。 与分片节点一样，DS 节点在每个 DS 时期结束时都会获得 $ZIL 奖励。

网络中 DS 节点的集合称为 DS 委员会。 与每个 DS 时期都完全修改的分片不同，DS 委员会在几个 DS 时期内保持相对稳定，因为每个 DS 时期只有一小部分委员会被替换。 DS 委员会成员由更高难度的工作量证明决定。 DS 节点在委员会内的任期长短也受其在 DS 时期内的表现的影响。

## 查询节点

查询节点是全节点，即存储 DS 和分片节点生成的所有区块链数据（包括交易、微块、交易块和 DS 块）。 查询节点通过 JSON-RPC 请求使最终用户可以使用这些数据。 它们还向正在同步或尝试加入网络的节点提供数据。 最后，查询节点还将交易分派给分片和 DS 委员会。

## 种子节点

种子节点（也称为档案节点）本质上是查询节点，只是它们不直接将交易分派给分片和 DS 委员会。 它们主要用于服务 JSON-RPC 请求，包括创建新交易。 交易由种子节点定期捆绑并转发到查询节点，以最终分派给分片和 DS 委员会。

## 质押种子节点

质押种子节点 (SSN) 是种子节点的一种变体。 SSN 由质押合作伙伴运行，并有望为来自社区的 JSON-RPC 请求提供服务。 作为服务的回报，这些节点会定期以 $ZIL 的形式获得质押奖励。质押奖励是根据 SSN 的表现确定的。 有关质押种子节点的更多信息，请参阅 [Zilliqa 改进提案 #3](https://github.com/Zilliqa/ZIP/blob/master/zips/zip-3.md) 中的质押功能文档。
